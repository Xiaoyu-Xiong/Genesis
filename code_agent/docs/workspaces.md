# Workspaces

`code_agent/workspaces/` will contain generated run workspaces.

Run workspaces can become large. Commit only curated examples or small fixtures intentionally.

## Standard Layout

```text
code_agent/workspaces/<run_id>/
  inputs/
    user_prompt.md
    repo_rules.md
    capabilities.md
  contracts/
    planner_output.json
    timing.json
    episode_plan.json
  assets/
    asset_manifest.json
  src/
    scene.py
    body.py
    action.py
    rendering.py
    main.py
  logs/
    codex_planner_turn_000.jsonl
    codex_planner_turn_000.final.json
    codex_scene.jsonl
    codex_scene.final.json
    codex_body.jsonl
    codex_body.final.json
    codex_action.jsonl
    codex_action.final.json
    codex_rendering.jsonl
    codex_rendering.final.json
    codex_critic.jsonl
    codex_critic.final.json
  artifacts/
    run_result.json
    event_log.json
    metrics.json
    render_stats.json
    render.mp4
    frames/
  reports/
    episode_state.json
    planner_actions.jsonl
    dispatch_history.jsonl
    worker_dispatch.json
    execution_report.json
    stdout.txt
    stderr.txt
    artifact_evaluation.json
    critic_report.json
    codex_critic_report.json
  summary.json
```

The current rigid pipeline writes `summary.json`, `metrics.json`, `event_log.json`, `run_result.json`, and, when
enabled, render artifacts under each case's `artifacts/` directory, plus suite-level `summary.json` at the run root
after a suite completes. `src/rendering.py` is generated by the dedicated Rendering Worker.

Worker logs include both `codex_<role>.jsonl` and `codex_<role>.final.json`. For current writers, the final message is a
JSON report describing the file edited, exports provided, assumptions, and remaining risks. The source files under
`src/` are edited directly by the assigned Codex writer in `workspace-write` sandbox mode.
